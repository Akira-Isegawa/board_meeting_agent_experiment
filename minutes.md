# 会議概要
- 新規事業「判断テーマ特化型ハッカソン事業」立ち上げに関し、経営・運営上のリスク、営業・現場推進、サービス設計、法務・会計まで幅広く討議。
- 目的は、コンサルや調査外注の前段を代替し、大企業の「決められない」意思決定を継続的に支援する判断市場を構築すること。

# 参加者
- 社長
- 営業担当役員
- 企画・設計担当役員
- 製造担当役員
- バックオフィス担当役員
- 製造業のコンサルタント
- 知的財産権の専門家
- 法務の専門家
- 会計の専門家

# 主な論点
- **成果/KPI設計**：意思決定品質の定性・定量評価、新規性の説明力（社長）
- **買い手獲得/営業戦略**：コンサル慣れ層・大企業稟議突破の難しさ、ターゲット業界・営業チャネルと失注・納得の根拠提示（営業担当役員）
- **パイロット案件・テーマ選定**：意思決定困難度、案件リスト精度とプロセス標準化・特化UX（企画担当役員）
- **製造現場フィット・UX**：現行会議体との重複/摩擦、アウトプット移植性、現場納得・カスタマイズ設計（製造担当役員）
- **バックオフィス/購買稟議/ガバナンス**：新サービス説明やIT・情報管理、購買部門の稟議・運用障壁、標準資料テンプレート化（バックオフィス役員）
- **既存会議型/コンサルとの差違**：成果物・判断スピード・中立性・納得感等の可視化、現場納得向上施策（製造コンサル）
- **知財・特許案件運用とリスク**：高度専門性・開示範囲・成果物の取扱い・秘密保持・後紛争リスク（知財専門）
- **法務的リスク設計**：多者間NDA、成果物・討議データの権利帰属、法的位置付けとガイドライン、クラウド基準（法務専門家）
- **会計/収益モデル・税務リスク**：単発受注・サブスク化、収益認識・成果物検収基準・費用配分・契約整備（会計専門家）

# 合意事項
- 初期パイロットは「意思決定困難×既存外注費高騰×経営層コミット」が明確な製造系案件（例：新素材採用/研究テーマ継続）を対象とする。
- 買い手・売り手両方に“参加価値”が納得できるよう、UX/アウトプット粒度や動機付け設計は各テーマ特化型で進める。
- パイロット案件獲得・現場フィードバック・営業資料は、「経営/現場双方が稟議しやすい」標準雛形・QA・契約・ITガバナンス対応を揃えて先行準備。
- 秘密保持、多者間NDA、データ管理、成果物の法的位置付け・用途限定等はパイロット前に厳密設計し、法務・知財両面でガイドライン化する。
- 年度・半期単位の継続課金/サブスク型契約、収益認識・契約整備・経費/原価管理など将来スケールへ向けた会計仕様化準備を開始。

# 未決事項
- パイロット候補先の具体リスト、営業実働状況、現場ヒアリング結果の最終化（業界/案件/部門/関係者名付き）。
- 既存コンサルとの差別化の現場/顧客“生の声”および比較データ取りまとめ、営業/バックオフィス/会計部との認識統一。
- ハッカソン成果物（論点マップ等）の社内移植・現場活用プロセス事例と、実証済み“納得感”取得例のストック化。
- 知財/特許テーマ案件の運用標準・秘密保持/成果物帰属の合意形成状況（NDA/契約雛形等）の現場回答待ち。
- 討議ログ・成果物等のデータ管理体制（保存・削除・アクセス管理等）の実態と社内ガバナンス基準適合調査の継続。
- 売り手側のメリット（営業機会/インサイト指標等）設計に現場営業/PE/PMO巻込み案の具体化。

# 次のアクション
- 各部門持ち回りで、パイロット候補企業と対象案件について現場・営業・企画部が合同で実地ヒアリングし、案件事例集と現場ペイン・決定遅延エビデンスを収集。
- 企画・営業主導で「パイロット提案用標準資料（稟議用雛形、QA、IPA/バックオフィス説明、データ管理ガイドライン）」を整備し、営業現場・顧客でリファイン。
- 法務・知財部が中心となり、マルチパーティNDA/成果物帰属契約、機密管理ルール、プロセスガイドラインのドラフトを作成。
- 会計部門で初期案件を想定した会計処理・原価配賦・収益認識プロセス・契約分類案を作成し、専門家レビューを実施。
- パイロット後、現場・顧客・営業のフィードバックを収集しモデル修正、再現可能性・拡張性評価を実施。KPI（再利用意向・納得度・成果リードタイム等）を追跡測定する仕組みも同時構築。
- 次回会議予定：各アクション進捗と初期パイロット案件の進展・現場/顧客評価・未解決課題の洗い出し状況を集約し、判断。
